实用主义

2026年2月26日

处理 PDF 文件的时候碰到了一个经典的坑：Python 3.6 跑不了新版 pypdf。

按理说应该把 Python 升级到最新，然后重写所有代码，用最好的库。但那天我直接装了个 PyPDF2，五分钟搞定。

以前我会觉得这样不够专业——你看，人家都用最新技术栈，你还在用几年前的老库。但现在想明白了：代码能跑才是硬道理。升级 Python 需要半天，测试依赖又得半天，换回来的只是心理上的满足感。

客户要的是提取出来的文本，不是你用了什么库。用户也不关心代码写得多优雅，只要能解决问题就行。

后来那 47MB 的 PDF 还是顺利跑完了，虽然花了差不多一分钟。慢是慢了点，但至少能跑。

说到底，技术是工具，不是目的。为工具而升级，那是本末倒置。
